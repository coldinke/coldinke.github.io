Some basic concepts

Keyword：
  1. struct
    • struct 默认的访问级别是 public
    • struct 默认的继承方式是 public
  2. class
    • class 默认的访问级别是 private
    • class 默认的继承方式是 private

在 C 中使用结构体时需要加上 struct 关键字，而在 C++ 中则不需要

  1. inline

    • inline: 在类内部定义的函数默认是 inline 的
    • inline: 修饰的函数在编译时会被编译器直接替换到调用处，而不是像普通函数一样调用
    • 注意：inline 仅建议编译器内联

  2. static

    • 普通的 static 变量，声明/定义通常在一起发生（不包括扩展static 生命周期的情况）
    • 对于 static 成员变量，类的里面是声明，类的外面是定义。
    • 调用 static 成员函数的方式有两种：
      1. 通过 object 调用
      2. 通过 CLASS NAME 调用
temp object

临时对象，其生命周期只有使用/声明的那一行。

Syntax: typename(Arguments)

在 C++ 中，临时对象绝对不可以返回其引用，因为其生命周期只有一行，当函数返回时，其所在的栈帧会被释放，故返回的引用会指向一个不存在的对象。

Life time for variables

生命周期

  1. stack object 其生命在作用域（scope）结束后自动结束。这种作用域内的 object，又称为 auto object。
  2. static local objects 其生命在作用域（scope）結束之後仍然存在，直到整個程序結束
  3. global objects
    • 其生命在整個程序結束之後才結束
    • 你也可以把它視為一種 static object，其作用域是「整個程序」
  4. heap objects 其生命在它被 deleted 之际结束
About new && delete operations

  1. new operation 实际可以等价于以下三条语句
String *ps = new String("Hello");
// |
// v
void *mem = operator new( sizeof(String) );  // 在 Heap 上分配内存
ps = static_cast<String*>(mem);       // 对数据类型进行转换（一般 分配器分配的内存地址类型为 void* 
ps->String::String("Hello");          // 调用对应类型的构造函数
  1. delete operation 也等价于以下两条语句
delete ps;
// |
// v
String::~String(ps);                  // 析构函数
operator delete(ps);                  // 释放内存
  1. 特殊的 new AND delete operations

在为 array 分配连续的内存空间时，会采用 new CLASS_NAME[NUM] 的方式来分配连续的内存空间，其中每个“元素”的空间容量一致。注意在释放时也要使用特殊的释放操作delete[] VARIABLE_NAME 才能正确的释放new [] 分配出的内存。

其中分配给 array 的内存能正确被释放，但因为普通的 delete 操作只会调用一次析构函数，会导致在 构造函数中分配的内存产生泄漏。

  1. new expression AND operator new

前面的 new 和 delete 是作为表达式，其中分解后的动作为 operator new

new 和 delete 作为表达式，不可以重载，但类类部的 new 和 delete 作为操作符，可以重载。

void *operator new(size_t size) {

struct Bad: exception
{
};

struct Foo {
    int id_{};

    Foo() = default;

    explicit Foo(int id): id_{id} { throw Bad(); }

    virtual ~Foo() = default;

    static void *operator new(size_t size) {
        ::printf("member operator new with size %zu\n", size);
        return ::malloc(size);
    }

    static void *operator new[](size_t size) {
        ::printf("member operator new[] with size %zu\n", size);
        return ::malloc(size);
    }

    static void operator delete(void *ptr) {
        ::printf("member operator delete with ptr %p\n", ptr);
        ::free(ptr);
    }

    static void operator delete[](void *ptr) {
        ::printf("member operator delete[] with ptr %p\n", ptr);
        ::free(ptr);
    }

    // placement new functions
    static void *operator new(size_t size, void *start) {
        ::printf("member operator new with size %zu start %p\n", size, start);
        return start;
    }

    static void *operator new(size_t size, size_t extra) {
        ::printf("member operator new with size %zu extra %zu\n", size, extra);
        return ::malloc(size + extra);
    }

    // placement delete functions
    static void operator delete(void *ptr, void *start) {
        ::printf("member operator delete with ptr %p start %p\n", ptr, start);
    }

    static void operator delete(void *ptr, size_t extra) {
        ::printf("member operator delete with ptr %p extra %zu\n", ptr, extra);
    }
};

int main() {

    {
        int *foo = new int;
        delete foo;
    }

    {
        Foo *foo = new Foo;
        printf("size: %zu\n", sizeof(Foo)); // roundup(4 + 8, 8);
        delete foo;
    }

    {
        Foo *foo = new Foo[4];
        delete[] foo;
    }

    {
        Foo *foo = ::new Foo[4]; // global
        ::delete[] foo;
    }

    {
        void *ptr = ::malloc(sizeof(Foo));
        // placement new expresion
        Foo *foo = new(ptr) Foo;
        // not placement delete expression
        delete foo;
    }

    {
        void *ptr = ::malloc(sizeof(Foo));
        try {
            Foo *foo = new(ptr) Foo(1);
        } catch(Bad &bad) {
            ::printf("%s\n", bad.what());
        }
    }

    {
        try {
            Foo *foo = new(4) Foo(1);
        } catch (Bad &bad) {
            ::printf("%s\n", bad.what());
        }
    }
}

注意，无法直接调用 placement delete function, placement delete function 仅用于当调用 placement new function 在其使用构造函数构造其内存结构发生异常时，为了防止内存泄漏时，才会调用 placement delete function

另外， 对于 operator delete 的重载，其 size_t 的参数为可选。

member functions

一个类里的函数可分为两类：

  • const member functions: 一定不会改变类中的数据成员，使用 const 关键字修饰
  • non-const member functions: 可能会改变类中的数据成员, 不使用 const 关键字修饰

示例：

对于一个 const 类型的数据（其类的设计者并没有提供 const overloading version），则会在编译过程中出现如下错误：

complex_test.cpp:7:13: error: 'this' argument to member function 'real' has type 'const complex', but function is not marked const
    cout << c1.real() << endl;
            ^~
./complex.h:11:12: note: 'real' declared here
    double real() { return re; }

故这类问题需要设计人员的必须保证的点，否则会出现 undefined behavior

补充：

When const and non-const versions of a member function are both available:

  • const object only call const version
  • non-const object only call non-const version

When only the const version or the non-const version of a member function is available:

  • const object only call const version (You can refer to the example above)
  • non-const object call non-const version and const version

The keyword const is part of the function signature, allowing you to overload a function by adding or removing the const keyword.

操作符重载

在 C++ 中可以视操作符为一个函数，对于几种常用的操作符做一些解析：

  • 二元操作符号： X + Y -> typename operator + (const typename& lhs, const typename& rhs)
  • 一元操作符：- Y -> typename operator - (const typename& rhs )

对于类似于以下的函数设计，不能 return by value 的原因是在其最后的结果是个 Local variable/object，当退出函数的作用域后其所在的栈帧会被释放。

  1. Operator overloading (non-member function) No this
ostream& 
operator << (ostream& os, const complex& x)
{
	return os << '(' << x.real() << ',' << x.imag() << ')';
}
  1. Operator overloading (member function) Have this
inline complex&
__doapl(complex* ths, const complex& r)
{
	ths->re += r.re;
	ths->im += r.im;
	return *ths;
}

inline complex&
complex::operator += (const complex& r)
{
	return __doapl(this, r);
}

从以上的例子来看常见的操作符重载可分为以下两类：

  • 成员函数(存在隐式 this 指针，指向当前对象的“右边”部分)
  • 非成员函数(不存在隐式 this)

在对 ostream 进行操作时，其返回值为 ostream& 的原因是为了实现 cout << a << b << c 的链式操作 其他测试并没有发现必须 return by reference 的情况

Object Oriented Programming (Object Oriented Design)

常见的三种设计范式：

  • Inheritance (继承)

Inheritance

The capability of a class to derive properties and characteristics from another class is called Inheritance.

为了能正确的调用对象的析构函数，一般要求具有层次结构的顶级类定义其析构函数为虚函数。因为在delete一个抽象类指针时候，必须要通过虚函数找到真正的析构函数

示例：

##include <iostream>

using namespace std;

class A 
{
public:
    A() { cout << "A::A(int)" << endl; }
    ~A() { cout << "A::~A()" << endl; }
};


class B: public A
{
public:
    B() { cout << "B:B(int)" << endl; }
    ~B() { cout << "B::~B()" << endl; }
};

int main() {
    A *ptrA = new B;

    delete ptrA;

    return 0;
}

// output
A::A()
B:B()
A::~A()

其正确的析构顺序应该为 ~B() -> ~A() 将 base class 的 析构函数定义为 virtual 就能得到预期的结果。

Inheritance 的抽象结构/内存结构

继承的方式：
  1. public: 将基类的公有成员变为自己的公有成员，基类的保护成员变为自己的保护成员
  2. protected: 将基类的公有成员和保护成员变为自己的保护成员
  3. private: 将基类的公有成员和保护成员变为自己的私有成员
继承体系下的构造和析构：

  1. 构造有内而外 Derived 的构造函数首先调用 Base 的 default 构造函数，然后才执行自己

  2. 析构由外而内 Derived 的析构函数先执行自己，然后才调用 Base 的析构函数

Inheritance with virtual
  • non-virtual 函数： 你不希望 derived class override 对应的函数
  • virtual 函数：你希望 derived class override 对应的函数，且你对它已有默认的定义
  • pure virtual 函数： 你要求 derived class 一定要 override 对应的函数，并且你对它没有默认的定义

当类模版可以作为模版参数时，我们在声明 stack 类模版只需要指定容器的类型而不用指定容器中元素的类型。

一个完整的示例：

##include <iostream>
##include <vector>
##include <deque>
##include <cassert>


template <typename T,
        template<typename Elem,
        typename = std::allocator<Elem>>
        class Cont = std::deque>
class stack {
private:
    Cont<T> elems; // elements
public:
    void push(T const &);
    void pop();
    T const &top() const;
    [[nodiscard]] bool empty() const { return elems.empty(); }

    template<typename T2,
            template<typename Elem2,
            typename = std::allocator<Elem2>>
            class Cont2>
    stack<T, Cont> &operator=(stack<T2, Cont2> const &);

    template<typename, template<typename, typename> class>
    friend
    class stack;
};

template<typename T, template<typename, typename> class Cont>
void stack<T, Cont>::push(T const &elem) {
    elems.push_back(elem);
}

template<typename T, template<typename, typename> class Cont>
void stack<T, Cont>::pop() {
    assert(!elems.empty());
    elems.pop_back();
}

template<typename T, template<typename, typename> class Cont>
T const &stack<T, Cont>::top() const {
    assert(!elems.empty());
    return elems.back();
}

template<typename T, template<typename, typename> class Cont>
template<typename T2, template<typename, typename> class Cont2>
stack<T, Cont> &stack<T, Cont>::operator=(stack<T2, Cont2> const& rhs) {
    elems.clear();
    elems.insert(elems.begin(),
                rhs.elems.begin(),
                rhs.elems.end());
    return *this;
}

int main() {
    stack<int> istack;
    stack<float> fstack;

    istack.push(1);
    istack.push(2);
    std::cout << "istack.top(): " << istack.top() << '\n';


    // manipulate float stack
    fstack.push(3.3);
    std::cout << "fstack.top(): " << fstack.top() << '\n';

    // assign stack of different type and manipulate again
    fstack = istack;
    fstack.push(4.4);
    std::cout << "fstack.top(): " << fstack.top() << '\n';

    // stack for doubles using a vector as an internal container
    stack<double, std::vector> vstack;
    vstack.push(5.5);
    vstack.push(6.6);
    std::cout << "vstack.top(): " << vstack.top() << '\n';

    vstack = fstack;
    std::cout << "vstack: ";
    while (!vstack.empty()) {
        std::cout << vstack.top() << ' ';
        vstack.pop();
    }
    std::cout << '\n';

}

Part 2: 语义方面

this pointer

  1. 类中的成员函数都有一个隐式的 this 指针，粗俗的说，this 指针指向当前对象的“右边”部分
  2. 对成员函数的调用会添加this-> 指针，从而产生Dynamic Binding的效果

注意： 在 operator overloading 中， 有两种情况：

  1. 一种是成员函数，此时 this 指针指向的是左边的对象
  2. 一种是非成员函数，此时并没有 this 指针，故需要显式的传入参数

Dynamic Binding and Static Binding

  • 指在对程序中的 class 对象进行操作时，其调用的函数是在运行时才确定的，还是在编译时就可以确定方法的调用。

  • 对于虚成员函数，调用时根据隐式的 this 指针找到对应的虚表中具体的调用目标（函数）相当于将函数调用翻译成 (*(p->vptr)[n])(p)

Dynamic Binding

动态绑定的解析路线（通过指针调用虚函数）

通过指针找到 virtual pointer，再通过 virtual pointer 找到 virtual table ，然后找到对应的函数。

Static Binding

  • 静态绑定是直接在 call 时，已经知道目标函数的地址，在之后完 call 后 PC 寄存器就已经存在再目标函数地址的区域。

  • 对于一般的非虚成员函数，其在“内存”中的地址是固定的，编译过程中编译器只需将函数译码为对应的 call + TARGET_ADDRESS 的指令即可。

Object Based vs. Object Oriented

  • Object Based: 面对的是单一 class 的设计
  • Object Oriented: 面对的是多重 classes 的设计， classes 和 classes 之间的关系

Access level

在 C++ 中使用以下关键字来修饰成员变量和成员函数的访问级别：

  1. public：
    • 一般来讲对于函数而言，我们需要对外界开放，所以一般定义为 public 级别的
  2. private：
    • 一般来讲对于数据而言，我们需要对外界隐藏，所以一般定义为 private 级别的
  3. protect：
    • 派生类的成员函数可以访问基类的保护成员

Arguments passing and return value passing

Arguments passing:

因为引用传递参数和值传递参数对使用者来说是透明的，所以使用者不需要知道接受者是以 reference 方式接受还是以 value 方式接受。

但在函数重载中，编译器无法区分两个函数的区别，即两个函数的函数签名相同，故不能同时存在

  • Pass by value

    • 如果声明中不存在符号 & 则表明使用 Value 传参 （从语义的角度来看是使用 Copy 来处理）

  • Pass by reference(to const)

    • 如果函数声明中存在 const 就是具体的 pass by reference to const
      • 对于 pass by reference to const 而言，其函数签名为： ReturnType FunctionName (const Type& Param) { }
    • 否则就是 pass by reference
      • 在函数内部对数据的操作会反映到调用者的数据上，故其函数签名为： ReturnType FunctionName (Type& Param) { }

返回值传递：

  • return by value
  • return by reference (to const)

与指针对比

  • 若返回值类型为 pointer，则必须返回一个 pointer
  • 若返回值类型为 reference， 可以返回一个 object， 也可返回一个 reference

conversion function 转换函数

将类类型转换为其他类型（基本类型、类类型）：

故其为成员函数，实际使用重载运算符来实现（存在 this 的操作符号重栽）

Syntax： operator TYPE() const { ... }

通常意义上 conversion function 不应该改变原数据的大小，只是单纯做类型转换（即改变其数据的存储结构）

示例：

// Fraction.h
class Fraction
{
public:
    Fraction(int num, int den): m_numerator(num), m_denominator(den) {}
    operator double() const {
        return (double)m_numerator / m_denominator;
    }
private:
    int m_numerator;
    int m_denominator;
};

// test_Example.cc
##include "Fraction.h"
##include <cassert>
 
// Use (void) to silence unused warnings.
##define assertm(exp, msg) assert(((void)msg, exp))

int main() {
    Fraction f(10, 5);
    double d = 4 + f;
    assert(d == 6.0);
    return 0;
}

reference

编译器在处理 reference 时实际是将它作为特殊的 pointer, 它具有如下特性：

  • 引用是被引用对象的一个“别名”
  • 引用一定要初始化
  • object 和其 reference 的内存空间大小相同，地址也相同（假象）
    • 为了 object 的行为保持一致
    • 使用 sizeof 和 &p 输出时会显示其所“指向”内容的 object
  • reference 通常不用于声明变量， 而用于参数类型(parameters type)和返回类型(return type) 的描述
  • 由于使用引用方声明的函数调用方式相同，故不能同时存在两个函数函数签名的区别只在是否传递方式为引用（返回），如若同时出现，编译器会出现 Ambiguity error

示例：

##include <iostream>

using std::cout, std::endl;

typedef struct LED {
    int r, g, b;
} LED;


int main(int argc, char *argv[]) {
    float x = 0;
    float *p = &x;
    float &r = x;

    cout << sizeof(x) << endl;
    cout << sizeof(p) << endl;
    cout << sizeof(r) << endl;

    cout << p << endl;
    cout << *p << endl;
    cout << x << endl;
    cout << r << endl;
    cout << &x << endl;
    cout << &r << endl;

    LED a1;
    LED &r1 = a1;

    cout << sizeof(a1) << endl;
    cout << sizeof(r1) << endl;
    cout << &a1 << endl;
    cout << &r1 << endl;

    return 0;
}
