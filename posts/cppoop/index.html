简述面向对象编程

面向对象编程这种范式是以**对象(Object)**为核心概念的一种编程范式，它以符合人类的思维方式和适合开发维护大型、复杂的程序而流行。许多语言最开始也基于这一编程范式而设计的，如 C++, Java, Python 等。

但 C++ 不单单只支持面向对象这一种编程范式，你可以用它编写出命令式、函数式等多种风格的代码。如果你阅读过 Scott Meyers 的 《Effective C++》 你应该知道下面这条建议：

View C++ as a federation of languages

在长久的发展中，C++ 没有了最初的那份纯粹，它多了许多复杂而有趣的特性，如funcaional, template 等。我想这或许是网上流传 C++ 学习曲线极高的缘由。说远了，我们说会本篇的主题 - 面向对象编程

正如前文所说面向对象的核心在对象，它由两部分组成一是属性(data)用于描述对象具备的特征（质）；二是方法(code)用于描述对象的行为。在此基础上出现了封装、类、继承、多态等概念。下面我们来思考个问题如何使用面向对象的思想来描述一个学生？

使用 C++ 写面向对象风格的代码

C++ 提供的机制/支持

既然说所有支持面向对象范式的语言都存在封装、继承和多态这三个显著特点，我们就从这三个特点来说说 C++ 提供了什么样的机制？

  • 封装
    • C++ 提供 class 关键字对具有相同属性和方法对象的共同描述，并且提供了 private 等关键字支持对 class 外部访问内部数据和方法的权限进行控制的方式。
    • 提供两种特殊的成员方法分别是 constructor 和 destructor 用于 class 在不同情况下实体化一个新对象和销毁对象的方法。

乍一看单独划分出 constructor 和 destructor 是无意义的，但实际上不然在一个存在继承和其他关系的 class 中其初始和销毁也不是一件简单的事情。

  • 继承
    • C++ 提供对类与类之间的继承关系实现 - 既基类和派生类。
    • 同 class 的机制一样，也提供了访问控制。如派生类只可以访问基类的 public 的属性和成员。
  • 多态
    • C++ 提供 virtual 关键字可以将方法定义为 virtual。 在继承体系下，如果基类对某一方法的定义为 virtual， 那么在派生类中需要对基类中的 virtual 方法进行重载。
      • 抽象类：不可以被实例化的类，可以作为接口来约束派生类
      • 纯虚函数： 没有具体实现的虚函数，在 C++ 中一般用 virutal TYPE NAME(Parameters) const = 0 来表示，抽象类就是拥有纯虚函数的 class。
    • 重载
      • 函数重载：用于对已有的函数根据需求进行重实现。
      • 操作符重载： 一种特殊的函数重载形式，C++ 中的操作符运算背后调用的其实是对应形式的函数，用于描述不同对象的同一种行为具体实现的不同。
